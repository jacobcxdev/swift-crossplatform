### Detail

**P2 — NavigationStack Binding-driven push (Phase 10)**
`_TCANavigationStack.androidPath.set` has an empty push branch (line 243-247). NavigationPath doesn't expose elements, so user-driven push via NavigationLink(state:) doesn't dispatch `store.send(.push(...))`. Reducer-driven push (state.path.append) works correctly via observation. Pop works in both directions.

**P2 — JVM type erasure for multi-destination (Phase 10)**
Single-destination NavigationStack is safe. Multi-destination apps where multiple `navigationDestination(for:)` calls register different `StackState<X>.Component` types would collide on JVM. Mitigation: type-discriminating destinationKey. Not blocking current apps.

**P2 — Dismiss JNI timing (Phase 10)**
Dismiss mechanism is architecturally complete (PresentationReducer wires on all platforms, DismissEffect has correct fallback). Integration tests show dismiss action delivery fails under full JNI effect pipeline timing. withKnownIssue wrappers in place.

**P3 — Gated TCA extensions (Phase 13)**
4 Binding+Observation blocks, Alert/ConfirmationDialog extensions, and IfLetStore excluded on Android. All have working alternatives (PresentationReducer path, modern @Observable pattern). Enabling requires conditional SkipFuseUI import — significant refactor for deprecated/alternative-available code.

**Infrastructure — Robolectric pipeline (Phase 11)**
`skip test` (Robolectric) blocked by skipstone symlink issue with local fork paths. `skip android test` (emulator) is the working canonical pipeline. Resolves when forks are published upstream.

**Infrastructure — swiftThreadingFatal stub**
Required until Swift 6.3 ships upstream fix (swiftlang/swift#77890). Auto-removal version-gated.
