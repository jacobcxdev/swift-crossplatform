**EphemeralState.swift (line 17):**
Remove `&& !os(Android)` from `#if canImport(SwiftUI) && !os(Android)`. Make conformance unconditional under `#if canImport(SwiftUI)`:

```swift
#if canImport(SwiftUI)
@_documentation(visibility: private)
extension AlertState: _EphemeralState {}

@_documentation(visibility: private)
@available(iOS 13, macOS 12, tvOS 13, watchOS 6, *)
extension ConfirmationDialogState: _EphemeralState {}
#endif
```

This is a single-line change. Research confirmed safe: AlertState and ConfirmationDialogState are pure Swift types with no SwiftUI dependency. The conformance enables PresentationReducer's auto-dismiss logic (ephemeralType(of:) returns non-nil, causing state to nil after button action).

**Popover.swift:**
Replace the entire-file `#if canImport(SwiftUI) && !os(Android)` guard with a platform split. Keep existing Darwin implementation unchanged. Add Android fallback that delegates to `.sheet(store:content:)`:

```swift
#if canImport(SwiftUI)
import SwiftUI

#if !os(Android)
// ... existing Darwin popover implementation (unchanged) ...
#else
// Android: popover falls back to sheet (Material3 bottom sheet)
// Per D2: Android has no native popover concept
@available(iOS 14, macOS 13, tvOS 17, watchOS 10, *)
extension View {
  public func popover<State, Action, Content: View>(
    store: Store<PresentationState<State>, PresentationAction<Action>>,
    attachmentAnchor: PopoverAttachmentAnchor = .rect(.bounds),
    arrowEdge: Edge = .top,
    @ViewBuilder content: @escaping (_ store: Store<State, Action>) -> Content
  ) -> some View {
    self.sheet(store: store, content: content)
  }
}
#endif
#endif
```

Signature must match existing Darwin version exactly (same parameters including attachmentAnchor and arrowEdge which are accepted but ignored on Android).

**Reducers to define (file-scope):**

1. `StackFeature` — Stack navigation reducer:
   - `@ObservableState struct State { var path = StackState<Path.State>() }`
   - `enum Action { case path(StackActionOf<Path>) }`
   - `@Reducer enum Path { case detail(DetailRow) }` where `DetailRow` is a simple reducer with `@ObservableState struct State { var title: String }` and `enum Action { case titleChanged(String) }`
   - Body: `Reduce { state, action in .none }.forEach(\.path, action: \.path)`

2. `PresentFeature` — Presentation reducer:
   - `@ObservableState struct State { @Presents var child: ChildSheet.State? }`
   - `enum Action { case child(PresentationAction<ChildSheet.Action>), showChild }`
   - `ChildSheet`: simple reducer with `@ObservableState struct State { var text: String }` and `enum Action { case setText(String) }`
   - Body: reduces `.showChild` → sets `state.child = ChildSheet.State(text: "")`, then `.ifLet(\.$child, action: \.child) { ChildSheet() }`

3. `AlertFeature` — Alert/dialog reducer:
   - `@ObservableState struct State { @Presents var alert: AlertState<Action.Alert>?, @Presents var dialog: ConfirmationDialogState<Action.Dialog>? }`
   - `enum Action { case alert(PresentationAction<Alert>), dialog(PresentationAction<Dialog>), showAlert, showDialog; enum Alert: Equatable { case confirmDelete }; enum Dialog: Equatable { case optionA, optionB } }`
   - `.ifLet(\.$alert, action: \.alert)` and `.ifLet(\.$dialog, action: \.dialog)`

4. `DismissChild` — Dismiss dependency reducer:
   - `@Dependency(\.dismiss) var dismiss`
   - `enum Action { case doneTapped }`
   - `.doneTapped` returns `.run { _ in await self.dismiss() }`

5. `EphemeralParent` — Parent hosting DismissChild + AlertFeature for @ReducerCaseEphemeral/@ReducerCaseIgnored:
   - `@Reducer enum Destination { @ReducerCaseEphemeral case alert(AlertState<AlertAction>); @ReducerCaseIgnored case ignored; case child(DismissChild) }`
