  <action>
  Fix the three-layer cascading failure that causes `reportIssue()` to silently write to stderr instead of failing tests on Android.

  **Layer 1 fix -- `isTesting` detection (`IsTesting.swift`):**

  The current `isTesting` property checks Xcode-specific environment variables (`XCTestBundlePath`, `XCTestConfigurationFilePath`, etc.) and Darwin-specific process names (`xctest`, `.xctest`). Skip's `skip android test` sets none of these.

  Add an `#if os(Android)` branch to the `isTesting` property that detects the Android test context. Two approaches in priority order:

  **Approach A (preferred -- environment inspection):**
  First, inspect what Skip actually sets by adding a temporary diagnostic print in the test setup. Based on Skip's test infrastructure, the test runner likely uses swift-corelibs-xctest which sets identifiable process arguments or environment variables.

  Add to the `isTesting` computed property:
  ```swift
  #if os(Android)
  // Skip's test runner uses swift-corelibs-xctest on Android.
  // Detect test context via:
  // 1. Process arguments containing "xctest" (swift-corelibs-xctest pattern)
  // 2. Or presence of XCTest module symbols
  if ProcessInfo.processInfo.arguments.contains(where: { $0.hasSuffix("xctest") || $0.contains("XCTest") }) {
      return true
  }
  // Fallback: check if XCTest symbols are loaded (dlsym)
  if let handle = dlopen(nil, RTLD_LAZY),
     dlsym(handle, "XCTestCase") != nil {
      return true
  }
  #endif
  ```

  Place this BEFORE the existing Darwin-specific checks so it short-circuits on Android. The existing checks will continue to handle iOS/macOS.

  **Approach B (fallback -- if process inspection doesn't work):**
  Register a custom `IssueReporter` at test launch that wraps the default behavior. This sidesteps `isTesting` entirely. Implement as a test setUp override in the fuse-library test infrastructure.

  **Layer 2 fix -- `dlsym` resolution (`SwiftTesting.swift`):**

  The `unsafeBitCast(symbol:in:)` function has `#if os(Linux)` / `#elseif canImport(Darwin)` / `#else return nil` branches. In Swift 6.2, `os(Android)` is distinct from `os(Linux)`. Android falls through to `return nil`, making all symbol lookups fail.

  Fix: Add `os(Android)` to the Linux branch since Android uses the same ELF dynamic linking:
  ```swift
  #if os(Linux) || os(Android)
    dlopen("lib\(library).so", RTLD_LAZY)
  #elseif canImport(Darwin)
    dlopen(nil, RTLD_LAZY)
  ```

  Apply this pattern to ALL `#if os(Linux)` guards in `SwiftTesting.swift` that deal with `dlopen`/`dlsym`. There may be multiple occurrences.

  Also check for any `#if canImport(Darwin)` fallback paths that should include Android. The pattern should be:
  - `#if os(Linux) || os(Android)` for ELF-based dynamic linking
  - `#if canImport(Darwin)` for Mach-O dynamic linking
  - `#else return nil` for truly unsupported platforms (WASI, Windows)

  **Layer 3 -- Verify fallback paths:**
  Check `SwiftTesting.swift` for any `#if DEBUG && canImport(Darwin)` guards that gate direct dlsym into Testing/XCTest frameworks. If these are the ONLY path to test failure reporting on non-Darwin, they need an Android equivalent. Document any that exist but do NOT change them unless they block test failure reporting after Layers 1 and 2 are fixed.

  **Important constraints:**
  - All changes must use inline `#if os(Android)` guards -- no separate platform files
  - Preserve existing behavior for iOS/macOS/Linux/WASI -- only ADD Android branches
  - Match upstream error detail level and Apple severity behavior
  - Commit changes within the xctest-dynamic-overlay fork submodule
  </action>

  import IdentifiedCollections
  import Testing

  struct User: Identifiable, Equatable, Codable {
      var id: Int
      var name: String
  }

  // IC-01: IdentifiedArrayOf initializes from array literal
  @Test func initFromArrayLiteral() {
      let users: IdentifiedArrayOf<User> = [
          User(id: 1, name: "Alice"),
          User(id: 2, name: "Bob"),
          User(id: 3, name: "Charlie"),
      ]
      #expect(users.count == 3)
  }

  // IC-02: Subscript read by ID returns correct element in O(1)
  @Test func subscriptReadByID() {
      let users: IdentifiedArrayOf<User> = [
          User(id: 1, name: "Alice"),
          User(id: 2, name: "Bob"),
      ]
      #expect(users[id: 1] == User(id: 1, name: "Alice"))
      #expect(users[id: 2]?.name == "Bob")
      #expect(users[id: 99] == nil)
  }

  // IC-03: Subscript write nil removes element
  @Test func subscriptWriteNilRemoves() {
      var users: IdentifiedArrayOf<User> = [
          User(id: 1, name: "Alice"),
          User(id: 2, name: "Bob"),
      ]
      users[id: 1] = nil
      #expect(users.count == 1)
      #expect(users[id: 1] == nil)
      #expect(users[id: 2] != nil)
  }

  // IC-04: remove(id:) returns removed element
  @Test func removeByID() {
      var users: IdentifiedArrayOf<User> = [
          User(id: 1, name: "Alice"),
          User(id: 2, name: "Bob"),
      ]
      let removed = users.remove(id: 1)
      #expect(removed == User(id: 1, name: "Alice"))
      #expect(users.count == 1)
  }

  // IC-05: ids property returns ordered set of all IDs
  @Test func idsProperty() {
      let users: IdentifiedArrayOf<User> = [
          User(id: 3, name: "Charlie"),
          User(id: 1, name: "Alice"),
          User(id: 2, name: "Bob"),
      ]
      #expect(Array(users.ids) == [3, 1, 2])
  }

  // IC-06: Codable conformance when element is Codable
  @Test func codableConformance() throws {
      let users: IdentifiedArrayOf<User> = [
          User(id: 1, name: "Alice"),
          User(id: 2, name: "Bob"),
      ]
      let data = try JSONEncoder().encode(users)
      let decoded = try JSONDecoder().decode(IdentifiedArrayOf<User>.self, from: data)
