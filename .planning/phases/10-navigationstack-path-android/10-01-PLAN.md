Update CLAUDE.md with the following additions (preserve all existing content, add new sections and expand existing ones):

**New Gotchas (add to existing Gotchas section):**
- **Android builds can pass when running fails**: `skip android build` validates compilation only. `skip android test` (or running on emulator) is the real verification gate. Always run tests, not just builds.
- **Clean builds required after dependency changes**: After modifying any Package.swift, run `swift package clean` in the example directory before building. Stale build caches cause spurious errors.
- **`withTransaction` is `fatalError()` on Android**: skip-fuse-ui marks `withTransaction` as `@available(*, unavailable)`. All animated navigation/dismiss must use non-animated paths (plain `store.send()` instead of `store.send(_:animation:)`).
- **skip-fuse-ui NavigationStack IS generic**: Unlike skip-ui (non-generic), skip-fuse-ui's `NavigationStack<Data, Root>` matches SwiftUI's signature. TCA extensions may compile directly against it.

**New Section "## Environment Variables" (add after Platform Conditionals):**
Document the three key conditionals and their effects:
```
| Conditional | Where Set | Effect |
|-------------|-----------|--------|
| `#if os(Android)` | Compiler | Android-specific code paths (production code) |
| `#if SKIP_BRIDGE` | Package.swift `.define("SKIP_BRIDGE")` | Bridge-level observation wrappers in skip-android-bridge |
| `TARGET_OS_ANDROID` | SPM `Context.environment` | Conditional SPM dependency inclusion at package resolution time |
```

Rewrite the Makefile to support smart defaults that build/test both fuse-library and fuse-app by default. Preserve all existing targets but change their behavior:

**New variable:**
```makefile
EXAMPLES ?= fuse-library fuse-app
```

**When EXAMPLE= is set** (backwards-compatible): single-example mode, works exactly as before.
**When EXAMPLE= is NOT set**: iterate over EXAMPLES, running each target for each example.

Implementation pattern (per the research code example):
```makefile
EXAMPLES ?= fuse-library fuse-app

# If EXAMPLE is set, use single-example mode (backwards compatible)
ifdef EXAMPLE
EXAMPLE_DIR := examples/$(EXAMPLE)
build:
	cd $(EXAMPLE_DIR) && swift build
test:
	cd $(EXAMPLE_DIR) && swift test
# ... etc
else
# Smart defaults: iterate over all examples
build: $(foreach ex,$(EXAMPLES),build-$(ex))
build-%:
	cd examples/$* && swift build

test: $(foreach ex,$(EXAMPLES),test-$(ex))
test-%:
	cd examples/$* && swift test

android-build: $(foreach ex,$(EXAMPLES),android-build-$(ex))
android-build-%:
	cd examples/$* && skip android build

android-test: $(foreach ex,$(EXAMPLES),android-test-$(ex))
android-test-%:
	cd examples/$* && skip android test

skip-test: $(foreach ex,$(EXAMPLES),skip-test-$(ex))
skip-test-%:
	cd examples/$* && skip test

skip-verify: $(foreach ex,$(EXAMPLES),skip-verify-$(ex))
skip-verify-%:
	cd examples/$* && skip verify --fix

clean: $(foreach ex,$(EXAMPLES),clean-$(ex))
clean-%:
	cd examples/$* && swift package clean
	@echo "Cleaned examples/$*"
endif
```

Keep the `test-filter` target working with EXAMPLE= override (default to fuse-library if EXAMPLE not set).

Keep all submodule management targets unchanged (status, push-all, pull-all, diff-all, branch-all).
