<objective>
Create an Android-specific NavigationStack adapter in TCA's `NavigationStack+Observation.swift` that bridges TCA's `Binding<StackState.PathView>` into skip-ui's expected `Binding<[Any]>`, and enable `_NavigationDestinationViewModifier` on Android.

Purpose: skip-ui's NavigationStack is non-generic and force-casts path to `Binding<[Any]>?`. TCA's `extension NavigationStack where Data == PathView` cannot compile on Android because of this non-generic constraint. An Android-specific View struct + free function provides the same API surface without requiring generic extension on NavigationStack.

Output: Modified `NavigationStack+Observation.swift` with Android adapter code that enables `NavigationStack(path:root:destination:)` on Android.

**Guard 3 (line 150) — The main NavigationStack extension + `_NavigationDestinationViewModifier`:** Restructure this block. The current `#if !os(Android)` wraps BOTH the `extension NavigationStack` (lines 151-192) AND `_NavigationDestinationViewModifier` (lines 194-218). These need different treatment:

1. **`_NavigationDestinationViewModifier` (lines 194-218):** Move OUTSIDE the `#if !os(Android)` guard so it compiles on all platforms. This struct is needed by the Android adapter to register destinations. It has no platform-specific code — it uses `navigationDestination(for:)` and `store.scope(component:)` which are both available on Android.

2. **`extension NavigationStack` (lines 151-192):** Keep inside `#if !os(Android)` — this extension requires `where Data == StackState<State>.PathView` which cannot compile against skip-ui's non-generic NavigationStack.

3. **Add Android adapter AFTER the `#if !os(Android)` block** with a new `#if os(Android)` section containing:

   a. A **View struct** `_TCANavigationStack<State: ObservableState, Action, Root: View, Destination: View>` that:
      - Accepts `pathBinding: Binding<Store<StackState<State>, StackAction<State, Action>>>`, `root: Root`, `destination: @escaping (Store<State, Action>) -> Destination`, and source location params (`fileID`, `filePath`, `line`, `column`)
      - In its `body`, creates a `Binding<[Any]>` adapter:
        - **get:** Maps `store.currentState.path` (PathView) to `[Any]` by mapping each `Component` to `Any`
        - **set:** Compares `newPath.count` with `store.currentState.count`:
          - If increased and `newPath.last` can be cast to `StackState<State>.Component`, dispatches `.push(id:state:)` with the component's id and element
          - If decreased, dispatches `.popFrom(id:)` using `store.currentState.ids[newPath.count]` to identify the element being popped
      - Renders `NavigationStack(path: androidPath) { root.modifier(_NavigationDestinationViewModifier(...)) }`
      - The `_NavigationDestinationViewModifier` is applied to root (same pattern as iOS) to register `.navigationDestination(for: Component.self)`

   b. A **free function** with the same signature as the iOS `NavigationStack.init(path:root:destination:)`:
      ```swift
      public func NavigationStack<State: ObservableState, Action, Destination: View, R: View>(
          path: Binding<Store<StackState<State>, StackAction<State, Action>>>,
          @ViewBuilder root: () -> R,
          @ViewBuilder destination: @escaping (Store<State, Action>) -> Destination,
          fileID: StaticString = #fileID,
          filePath: StaticString = #filePath,
          line: UInt = #line,
          column: UInt = #column
      ) -> some View
      ```
      This returns `_TCANavigationStack(...)` with all parameters forwarded.

**IMPORTANT constraints:**
- Do NOT use `store.send(_:animation:)` — `withTransaction` is `fatalError()` on Android. Use plain `store.send(...)`.
- Do NOT enable the `ObservedObject.Wrapper` scope (Guard 1) — deprecated.
- Do NOT enable the `Perception.Bindable` scope (Guard 2) — type doesn't exist on Android.
- Mark both the struct and free function with `@available(iOS 16, macOS 13, tvOS 16, watchOS 9, *)`.
- The `NavigationLink(state:)` extensions (lines 264-359) are already OUTSIDE the guard and compile on all platforms — do not modify them.
- The `Store.scope(component:)` extension (lines 221-262) is already OUTSIDE the guard — do not modify it.
