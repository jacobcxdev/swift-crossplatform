  <name>Task 1: Validate SPM compilation for all 14 forks on Android</name>
  <files>

  <action>
  Validate that all 14 fork submodules compile for Android via Skip Fuse mode. This is a validation task, not an implementation task -- the SPM configurations already exist. But if compilation fails, fix the issues.

  **Step 1: Verify environment**
  Run `skip doctor --native` to confirm the Android SDK, NDK, and Swift Android toolchain are properly configured. If any check fails, follow the doctor output to resolve before proceeding.

  Run `skip devices` to list available Android emulators. At least one emulator must be available for `skip test` later.

  **Step 2: Verify submodules are initialized**
  Run `git submodule update --init --recursive` from repo root to ensure all 14 forks are checked out and up to date.

  **Step 3: Incremental build validation**
  Build in dependency order to isolate any failures. From `examples/fuse-library/`:

  Layer 1 (no fork dependencies):
  - `swift build` on macOS first (quick sanity check)

  Layer 2 (Android build):
  - `make android-build` (or `cd examples/fuse-library && skip android build`)
  - This resolves ALL 14 forks through the dependency graph and compiles for Android

  If `make android-build` fails:
  - Read the error output carefully
  - Common issues: missing `#if os(Android)` guards, SwiftSyntax macro compilation, unresolved symbols
  - Fix any issues in the failing fork's source files
  - Re-run the build

  **Step 4: Verify SPM-specific requirements**
  After successful Android build, verify these SPM requirements by inspecting the build output and Package.swift files:

  - **SPM-01**: Per CONTEXT.md hybrid pattern: Skip forks (skip-android-bridge, skip-ui) use `Context.environment["SKIP_BRIDGE"]` conditionals; Point-Free forks use `#if os(Android)`. Grep across `forks/*/Package.swift` for both `SKIP_BRIDGE` (Skip forks) and `os(Android)` (PF forks). Verify skip-ui and skip-android-bridge use `SKIP_BRIDGE`, not `TARGET_OS_ANDROID`.
  - **SPM-02**: Check that library products use `type: .dynamic` where needed for Skip Fuse mode. Grep for `.dynamic` in `forks/*/Package.swift`.
  - **SPM-03**: Check that Skip-processed targets have `.plugin(name: "skipstone", package: "skip")`. Grep for `skipstone` in relevant Package.swift files.
  - **SPM-04**: Verify macro targets (e.g., in swift-composable-architecture, swift-case-paths) compile. These run at compile time on the host, so they should work regardless of target platform. The Android build succeeding confirms this.
  - **SPM-05**: Verify `.package(path: "../../forks/<name>")` local path overrides in `examples/fuse-library/Package.swift` resolve correctly. The build succeeding confirms this.
  - **SPM-06**: Verify `swiftLanguageModes` and `swiftSettings` with `.define()` propagate. Check the build log for the expected compiler flags.

  **The 14 forks to verify (all must compile):**
  1. skip-android-bridge
  2. skip-ui

  4. swift-perception

  <action>
  Run the full observation test suite on both macOS (regression check) and Android emulator (bridge validation). This is the definitive proof that the observation bridge works end-to-end.

  **Step 1: macOS regression check**
  ```bash
  cd examples/fuse-library && swift test
  ```
  ALL tests must pass. This confirms the bridge code changes from Plan 01-01 did not break iOS/macOS behavior.

  **Step 2: Android emulator test**
  ```bash
  make skip-test
  ```
  Or equivalently: `cd examples/fuse-library && skip test`

  This runs the test suite on Android emulator via Skip. The observation tests exercise:
  - `withObservationTracking` via native `libswiftObservation.so` on Android
  - `ObservationVerifier` methods run as native Swift via JNI bridge
  - The transpiled Kotlin test harness calls into native Swift and asserts results

  **If `skip test` fails:**
  - Check `adb logcat -s swift` for native Swift crash logs
  - Check for JNI resolution failures (missing `_cdecl` exports)
  - Check for `swiftThreadingFatal` symbol resolution
  - Fix any issues and re-run

  **Step 3: Verify observation bridge is active**
  After successful `skip test`, check the Android logcat output for:
  - Absence of "ViewObservation: nativeEnable() failed" error message (bridge loaded successfully)
  - If diagnostics were enabled in tests, check for diagnostic output

  **Step 4: Run skip verify**
  ```bash
  make skip-verify
  ```
  Or: `cd examples/fuse-library && skip verify --fix`

  This runs Skip's verification checks which may catch additional issues.

  **Step 5: Document results**
  Record which tests pass on Android vs macOS. All ObservationTests should pass on both platforms. ObservationTrackingTests are macOS-only (they use direct `withObservationTracking` which doesn't go through the bridge path).
