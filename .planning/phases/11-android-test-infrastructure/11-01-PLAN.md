**Part B: Gate non-transpilable test files with `#if !SKIP`**

Add `#if !SKIP` / `#endif` guards around the ENTIRE content of test files that use APIs not available in Skip's Kotlin transpilation. The skipstone plugin will attempt to transpile all Swift files in the target -- ungated files with Combine, native Observation, XCTest internals, or Apple-only frameworks will cause Kotlin compilation errors.

**Strategy:** Wrap ALL existing test files in the 6 newly-enabled targets with `#if !SKIP` initially. This is the safest approach -- it ensures `swift build` and `skip test` succeed immediately. Individual test files can be un-gated in Plan 03 after investigating which tests actually transpile cleanly.

For each test file (all 21 files listed in FoundationTests, TCATests, SharingTests, NavigationTests, DatabaseTests, FuseAppIntegrationTests):
1. Add `#if !SKIP` as the very first line
2. Add `#endif` as the very last line

This preserves all macOS test execution (the tests still run natively on Darwin) while preventing Kotlin transpilation errors. The `#if !SKIP` guard is stripped during Skip transpilation, so only the XCSkipTests.swift file will be visible to the Kotlin compiler.

**Important:** Do NOT gate ObservationTests files -- they already have appropriate `#if !SKIP` guards. Do NOT gate FuseAppTests -- it already works with skipstone.
