**Step 1: Replace both JUnit stubs with canonical XCGradleHarness**

Replace the contents of both XCSkipTests.swift files (fuse-library/ObservationTests and fuse-app/FuseAppTests) with the canonical pattern. These files currently write fake JUnit XML with `tests="0"` -- replace with real `XCGradleHarness`/`runGradleTests()`.

Use exactly this pattern (matching `examples/lite-app/Tests/LiteAppTests/XCSkipTests.swift`):

```swift
import Foundation
#if os(macOS)
import SkipTest

@available(macOS 13, macCatalyst 16, *)
final class XCSkipTests: XCTestCase, XCGradleHarness {
    public func testSkipModule() async throws {
        try await runGradleTests()
    }
}
#endif
```

Preserve the runtime detection helpers (`isJava`, `isAndroid`, `isRobolectric`, `is32BitInteger`) at file scope outside the `#if os(macOS)` guard -- other test files may reference them.

**Step 2: Diagnose the skipstone symlink resolution issue**

Run `skip test` from both example directories and capture the Gradle output. The research documents that local fork path overrides (`../../forks/`) break Gradle's Swift dependency resolution through skipstone symlinks. Diagnose:

1. Run `cd examples/fuse-library && skip test --verbose 2>&1 | tail -100` to capture Gradle errors
2. Inspect the skipstone plugin output directory structure: `ls -la examples/fuse-library/.build/plugins/outputs/fuse-library/ObservationTests/destination/skipstone/`
3. Check for `NO-SOURCE` markers in Gradle tasks or missing symlinks
4. If Gradle fails with dependency resolution errors, inspect the generated `settings.gradle.kts` and `build.gradle.kts` in the skipstone output to understand how local packages are being resolved

**Step 3: Apply mitigation based on diagnosis**

Based on the diagnosis in Step 2, apply the appropriate fix:

- **If symlinks are resolvable with path adjustments:** Fix the paths in the skipstone output configuration or provide Gradle `includeBuild` overrides.
- **If skipstone cannot resolve local forks at all:** Accept that `runGradleTests()` will throw for test targets with complex transitive dependency chains through local forks. In this case:
  - For targets where `runGradleTests()` throws, wrap the call in a do/catch that logs the error and marks the test as skipped (not failed) with a clear diagnostic message. Use `XCTSkipIf` or `try XCTSkipUnless(false, "skipstone cannot resolve local fork paths: \(error)")`.
  - Document the specific error and which targets are affected.
  - The fallback path is `skip android test` which uses a different build pipeline that handles local packages correctly.

**Important:** Do NOT fall back to JUnit stubs. Either `runGradleTests()` succeeds (ideal) or it throws with a diagnostic skip (acceptable). Zero fake results.
