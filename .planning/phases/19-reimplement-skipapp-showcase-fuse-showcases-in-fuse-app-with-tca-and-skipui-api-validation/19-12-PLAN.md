    Update ShowcaseFeature.swift to route all 84 playgrounds through the NavigationStack:

    **Approach:** Since most playgrounds are plain Views (no TCA reducer), use a single `PlaygroundPlaceholderFeature` case in ShowcasePath that wraps the PlaygroundType and renders the appropriate view. This avoids creating 84 separate reducer cases (anti-pattern noted in research).

    **Update ShowcasePath** to keep the single `.playground(PlaygroundPlaceholderFeature)` case. The destination view uses a switch on `store.type` to render the correct playground view.

    **Update ShowcaseView** navigation destination to switch on the playground type and render the corresponding view:
    ```swift
    .navigationDestination(store: store.scope(state: \.playground, action: \.playground)) { store in
        PlaygroundDestinationView(store: store)
    }
    ```

    Create `PlaygroundDestinationView` (can be in the same file) that:
    1. Takes `StoreOf<PlaygroundPlaceholderFeature>`
    2. Switches on `store.type` to render the correct playground view
    3. Sets `.navigationTitle(store.type.title)` on each destination
    4. Covers all 84 cases of PlaygroundType

    **The switch statement should map each PlaygroundType case to its View:**
    ```swift
    switch store.type {
    case .accessibility: AccessibilityPlayground()
    case .alert: AlertPlayground()
    // ... all 84 cases
    case .zIndex: ZIndexPlayground()
    }
    ```

    **Also update the list to use NavigationLink with proper state:**
    Each row should be `NavigationLink(state: ShowcasePath.State.playground(.init(type: playground)))` with a label showing the playground icon + title.

    If any playgrounds created TCA @Reducer wrappers (Claude's discretion in Plans 08-11), add dedicated cases in ShowcasePath for those reducers and scope to their stores in the destination. But most should route through the single `.playground` case.
