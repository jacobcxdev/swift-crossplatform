In ViewAction.swift, the `#if !os(Android)` block (lines 31-43) gates out `send(_:animation:)` and `send(_:transaction:)`. Replace this with a platform-split that provides no-op overloads on Android:

```swift
  #if !os(Android)
  /// Send a view action to the store with animation.
  @discardableResult
  public func send(_ action: StoreAction.ViewAction, animation: Animation?) -> StoreTask {
    self.store.send(.view(action), animation: animation)
  }

  /// Send a view action to the store with a transaction.
  @discardableResult
  public func send(_ action: StoreAction.ViewAction, transaction: Transaction) -> StoreTask {
    self.store.send(.view(action), transaction: transaction)
  }
  #else
  /// Send a view action to the store with animation (no-op on Android).
  @discardableResult
  public func send(_ action: StoreAction.ViewAction, animation: Animation?) -> StoreTask {
    self.store.send(.view(action))
  }

  /// Send a view action to the store with a transaction (no-op on Android).
  @discardableResult
  public func send(_ action: StoreAction.ViewAction, transaction: Transaction) -> StoreTask {
    self.store.send(.view(action))
  }
  #endif
```

This preserves the iOS behavior (delegating to store.send with animation/transaction) while providing compile-compatible overloads on Android that ignore the animation/transaction parameter. The `Animation` and `Transaction` types are available on Android via skip-fuse-ui's SwiftUI re-export.

Do NOT modify the `#if canImport(SwiftUI)` outer guard (line 1) -- it correctly gates the entire file.
Do NOT modify the plain `send(_:)` method (line 27) -- it already works on all platforms.
