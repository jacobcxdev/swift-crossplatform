Define inline test reducers (do NOT import from other modules — keep tests self-contained):

**Counter reducer** (for TCA-01, TCA-02, TCA-03):
```swift
@Reducer
struct Counter {
    struct State: Equatable { var count = 0 }
    enum Action { case increment, decrement }
    var body: some ReducerOf<Self> {
        Reduce { state, action in
            switch action {
            case .increment: state.count += 1; return .none
            case .decrement: state.count -= 1; return .none
            }
        }
    }
}
```

**Parent/Child reducer** (for TCA-04, TCA-05, TCA-09):
```swift
@Reducer
struct Parent {
    struct State: Equatable { var child = Counter.State() }
    enum Action { case child(Counter.Action) }
    var body: some ReducerOf<Self> {
        Scope(state: \.child, action: \.child) { Counter() }
    }
}
```

**Tests to write:**

1. `testStoreInitialState` (TCA-01): Store inits with correct state. Assert `store.withState(\.count) == 0`.

2. `testStoreInitWithDependencies` (TCA-02): Store init with `withDependencies:` closure. Override `\.uuid` to `.incrementing`, create store, verify dependency is accessible inside reducer by dispatching an action that uses it.

3. `testStoreSendReturnsStoreTask` (TCA-03): `store.send(.increment)` returns `StoreTask`. Assert state changed. Call `await storeTask.finish()`.

4. `testStoreScopeDerivesChildStore` (TCA-04): Create parent store, scope to child. Send action on child store. Assert parent state reflects change.

5. `testScopeReducer` (TCA-05): Parent with `Scope(state:action:)`. Send child action through parent store. Assert child state updates.

6. `testIfLetReducer` (TCA-06): Reducer with optional child state. Use `.ifLet(\.detail, action: \.detail)`. Send action when state is non-nil — child reducer runs. When nil — no crash.

7. `testForEachReducer` (TCA-07): Reducer with `IdentifiedArrayOf<Item>`. Use `.forEach(\.items, action: \.items)`. Send action targeting specific item by ID. Assert only that item changed.

8. `testIfCaseLetReducer` (TCA-08): Reducer with enum state (e.g., `.loading`, `.loaded(Data)`). Use `.ifCaseLet`. Send action when in correct case — runs. When in different case — no-op.

9. `testCombineReducers` (TCA-09): Multiple reducers composed via `CombineReducers`. Both run in sequence.

10. `testEffectSend` (TCA-16): `Effect.send(.action)` synchronously dispatches. Verify state change happens synchronously.

11. `testStoreWithState`: `store.withState { $0.count }` reads state correctly without observation tracking.

Use `import ComposableArchitecture` and `import XCTest`. Use `@Reducer` macro for all reducer definitions. Use `store.withState(\.keyPath)` for state assertions (NOT direct state access — Store state is internal).
