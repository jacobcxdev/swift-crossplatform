    static let testValue: Int = 0

        get { self[TestCounterKey.self] }
        set { self[TestCounterKey.self] = newValue }

**Tests to write:**

1. `testDependencyKeyPathResolution` (DEP-01): Inside `withDependencies { $0.testCounter = 99 } operation: { }`, access `@Dependency(\.testCounter)`. Assert it equals 99.

4. `testTestValueInTestContext` (DEP-04): In test execution, verify `@Dependency(\.context)` returns `.test`. The testValue should be used automatically in tests.

7. `testWithDependenciesSyncScoping` (DEP-09): `withDependencies { $0.uuid = .incrementing } operation: { }`. Inside operation, `@Dependency(\.uuid)` returns incrementing UUIDs. Outside, it returns random UUIDs.

8. `testPrepareDependencies` (DEP-10): `Store(initialState:reducer:withDependencies:)` — the `prepareDependencies` closure runs before any `@Dependency` access. Override `\.continuousClock` in the closure. Verify the reducer uses the overridden clock.

9. `testChildReducerInheritsDependencies` (DEP-11): Create parent store with dependency override. Scope to child. Child's `@Dependency` resolves the parent's override. Test with 2 levels AND 3 levels deep (grandparent -> parent -> child per context decision).

    - `\.calendar` — it's a Calendar
    - `\.timeZone` — it's a TimeZone
    - `\.locale` — it's a Locale
    - `\.context` — it returns a DependencyContext value

11. `testDependencyIsolationBetweenSiblings` (DEP-11, context decision): Create parent with two child scopes. Override a dependency in child A. Verify child B does NOT see child A's override. This tests that overrides don't leak between siblings.
