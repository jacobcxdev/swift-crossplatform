**Part A: Combine publisher tests (SharedObservationTests.swift)**

The current file has TWO problems preventing Android transpilation:
1. The ENTIRE file is wrapped in `#if !SKIP` (line 1) — skipstone ignores it completely
2. It uses Swift Testing (`import Testing`, `@Test`, `@Suite`, `confirmation()`) which skipstone cannot transpile

The file contains 8 tests: 4 Combine publisher tests (SHR-09/SHR-10) and 4 non-Combine tests (SHR-12/SHR-13). The non-Combine tests are covered by other test files, so the priority is the 4 publisher tests.

**Approach:** Restructure the file so that:
- The `#if !SKIP` wrapper is REMOVED from the entire file
- Swift Testing imports and decorators are gated with `#if !SKIP` (skipstone can't handle them)
- Publisher tests use XCTest format (XCTestCase subclass with `func test*()` methods) so skipstone can transpile them
- Publisher import guard uses `#if canImport(Combine) || canImport(OpenCombine)` (matching the library's own pattern) instead of just `#if canImport(Combine)`
- On Android, import `OpenCombineShim` (not `OpenCombine` directly) — this matches SharedPublisher.swift's pattern

Rewrite the file as follows:

```swift
#if canImport(Combine)
import Combine
#elseif canImport(OpenCombine)
import OpenCombineShim
#endif
import Foundation
#if !SKIP
import Testing
#endif
import Sharing
import XCTest

// MARK: - File-scope types

// MARK: - XCTest Publisher Tests (Android-transpilable)

#if canImport(Combine) || canImport(OpenCombine)
final class SharedPublisherTests: XCTestCase {

    @MainActor
    func testSharedPublisher() async throws {
        @Shared(.inMemory("pubTest")) var count = 0
        var received: [Int] = []
        let expectation = self.expectation(description: "publisher emits")

        let cancellable = $count.publisher
            .dropFirst()
            .sink { value in
                received.append(value)
                expectation.fulfill()
            }

        $count.withLock { $0 = 42 }

        await fulfillment(of: [expectation], timeout: 2.0)
        _ = cancellable
        XCTAssertTrue(received.contains(42), "Publisher should have emitted 42, got: \(received)")
    }

    @MainActor
    func testSharedPublisherMultipleValues() async throws {
        @Shared(.inMemory("pubMulti")) var count = 0
        var received: [Int] = []
        let expectation = self.expectation(description: "publisher emits 3 values")
        expectation.expectedFulfillmentCount = 3

        let cancellable = $count.publisher
            .dropFirst()
            .sink { value in
                received.append(value)
                expectation.fulfill()
            }

        $count.withLock { $0 = 1 }
        $count.withLock { $0 = 2 }
        $count.withLock { $0 = 3 }

        await fulfillment(of: [expectation], timeout: 2.0)
        _ = cancellable
        XCTAssertEqual(received, [1, 2, 3])
    }

    @MainActor
    func testPublisherValuesAsyncSequence() async throws {
        @Shared(.inMemory("asyncPub")) var count = 0
        var received: [Int] = []
        let expectation = self.expectation(description: "publisher completes")

        let cancellable = $count.publisher
            .dropFirst()
            .prefix(3)
            .sink(
                receiveCompletion: { _ in expectation.fulfill() },
                receiveValue: { received.append($0) }
            )

        $count.withLock { $0 = 10 }
        $count.withLock { $0 = 20 }
        $count.withLock { $0 = 30 }

        await fulfillment(of: [expectation], timeout: 2.0)
        _ = cancellable
        XCTAssertEqual(received, [10, 20, 30])
    }

    @MainActor
    func testPublisherAndObservationBothWork() async throws {
        @Shared(.inMemory("bothChannels")) var count = 0
        var publisherReceived = false
        let expectation = self.expectation(description: "publisher receives 7")

        let cancellable = $count.publisher
            .dropFirst()
            .sink { value in
                if value == 7 {
                    publisherReceived = true
                    expectation.fulfill()
                }
            }

        $count.withLock { $0 = 7 }

        await fulfillment(of: [expectation], timeout: 2.0)
        _ = cancellable
        XCTAssertTrue(publisherReceived)
        XCTAssertEqual(count, 7)
```

**Part B: Enable TextState formatting on Android (TextState.swift + TextStateButtonStateTests.swift)**

TextState.swift in `forks/swift-navigation/Sources/SwiftNavigation/TextState.swift` gates ALL formatting modifiers behind `#if canImport(SwiftUI) && !os(Android)`. Investigation reveals this is overly conservative:

**Available in SkipSwiftUI (skip-fuse-ui):** `Font`, `Font.Weight`, `Font.Design`, `Color`, `CGFloat`, `LocalizedStringKey` (Text/LocalizedStringKey.swift)
**Available in SkipFoundation:** `LocalizedStringResource` (LocalizedStringResource.swift)
**Defined locally in TextState.swift:** `FontWidth`, `LineStylePattern`, `AccessibilityTextContentType` (simple enums, not SwiftUI imports)

ALL types used by TextState are available on Android. Zero types need gating — the entire `#if canImport(SwiftUI) && !os(Android)` guard can be replaced with `#if canImport(SwiftUI) || canImport(SkipSwiftUI)`.

**Approach:** In `forks/swift-navigation/Sources/SwiftNavigation/TextState.swift`:

1. Add SkipSwiftUI import at the top alongside the existing SwiftUI import:
```swift
#if canImport(SwiftUI) && !os(Android)
  import SwiftUI
#elseif canImport(SkipSwiftUI)
  import SkipSwiftUI
#endif
```

2. Change the guards that protect the `Modifier` enum, the `modifiers` array, and the core formatting methods from `#if canImport(SwiftUI) && !os(Android)` to `#if canImport(SwiftUI) || canImport(SkipSwiftUI)` — BUT only for sections that use types available in SkipSwiftUI.

3. ALL types are available on Android: SkipSwiftUI provides `LocalizedStringKey` and SkipFoundation provides `LocalizedStringResource`. The `Modifier` enum, ALL modifier methods, `FontWidth`, `LineStylePattern`, `AccessibilityTextContentType`, and localised string storage are ALL safe to enable.

4. The ONLY sections that may need to keep the original guard are:
   - `LocalizedStringResourceBox` (uses `@unchecked Sendable` wrapper around `Any` — check if pattern works with SkipFoundation's `LocalizedStringResource`)
   - `LocalizedStringKey.formatted()` extension (uses reflection on SwiftUI internals — likely Apple-specific)
   - `Text.init(_ state:)` SwiftUI rendering (the existing Android implementation using `_plainText` provides the Android rendering path; the SwiftUI-specific rendering uses Apple's `SwiftUI.Text` APIs)

   For these, keep `#if canImport(SwiftUI) && !os(Android)` and provide Android alternatives where needed.

5. In TextStateButtonStateTests.swift, remove the `#if !os(Android)` guard around `textStateWithFormatting()` since `.bold()` and `.italic()` will now be available on Android.

**Fallback:** If enabling SkipSwiftUI imports in TextState.swift causes cascading compilation issues (e.g. swift-navigation's Package.swift doesn't depend on skip-fuse-ui and `canImport(SkipSwiftUI)` is false), document the specific blocker and keep the guard with an explanatory comment. The key insight to record: the types exist in SkipSwiftUI but the dependency graph may not expose them to swift-navigation.
