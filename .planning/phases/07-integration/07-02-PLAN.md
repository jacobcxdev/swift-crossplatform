**ObservationBridgeTests.swift (TEST-10 — Tier 1 macOS mock-bridge):**

Since the bridge code is behind `#if SKIP_BRIDGE` and can't compile on macOS, test the OBSERVATION SEMANTICS that the bridge must preserve, using native `withObservationTracking`:

1. **Single property mutation triggers exactly one onChange:** Create `@Observable class Model { var count = 0 }`. Wrap access in `withObservationTracking`, mutate `count` 5 times in sequence. Assert onChange fires exactly once (not 5 times). This validates the coalescing semantics the Android bridge must match.

2. **Nested observation scopes are independent:** Create parent and child `@Observable` models. Track parent in outer scope, child in inner scope. Mutate child — only inner onChange fires. Mutate parent — only outer onChange fires.

3. **Bulk mutations coalesce into single onChange:** Create `@Observable class Model { var a = 0; var b = 0; var c = 0 }`. Track all three in one `withObservationTracking` scope. Mutate a, b, c in sequence. Assert ONE onChange callback total.

4. **@ObservationIgnored suppresses tracking:** Create model with `@ObservationIgnored var debug = ""`. Access and mutate `debug` — onChange must NOT fire.

5. **ObservableState registrar round-trip:** Create an `@ObservableState struct` (from TCA), access properties through a Store, mutate via `store.send()`. Verify the observation registrar correctly tracks and fires changes (validates the TCA → Observation bridge path).

6. **Concurrent observation on multiple threads:** Spawn 10 tasks, each creating its own `withObservationTracking` scope on the same `@Observable` model. Mutate model from main thread. Assert all 10 scopes fire their onChange independently (validates thread-local stack isolation).

**StressTests.swift (TEST-11 — two separate tests per D10):**

**Test 1: Store/Reducer throughput (pure state management):**
```swift
import Testing

extension Tag {
    @Tag static var stress: Self
}

@Test(.tags(.stress))
func storeReducerThroughput() async {
    // Simple counter reducer with .none effect
    let store = Store(initialState: StressCounter.State()) {
        StressCounter()
    }
    let iterations = 5_000
    let clock = ContinuousClock()

    let startMem = currentResidentMemoryBytes()
    let elapsed = clock.measure {
        for _ in 0..<iterations {
            store.send(.increment)
        }
    }
    let endMem = currentResidentMemoryBytes()

    // Must complete — proves >1000 mutations/sec
    #expect(elapsed < .seconds(5))
    // Memory must be bounded
    if let s = startMem, let e = endMem {
        #expect(e < s + 50 * 1024 * 1024) // <50MB growth
    }
}
```

**Test 2: Observation pipeline under load:**
```swift
@Test(.tags(.stress))
func observationCoalescingUnderLoad() async {
    let model = StressModel() // @Observable class with 3 properties
    var changeCount = 0
    let iterations = 5_000

    for _ in 0..<iterations {
        withObservationTracking {
            _ = model.value
        } onChange: {
            changeCount += 1
        }
        model.value += 1
    }

    // Each cycle: track → mutate → one onChange. Total = iterations.
    #expect(changeCount == iterations)
    // Memory bounded — no unbounded accumulation
}
```

Include the cross-platform `currentResidentMemoryBytes()` helper function from research (R3b §3) using `mach_task_basic_info` on Darwin and `/proc/self/status` on Linux/Android.
