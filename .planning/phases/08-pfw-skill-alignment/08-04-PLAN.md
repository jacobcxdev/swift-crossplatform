Migrate each file from XCTestCase to Swift Testing. These are the simplest conversions (no expectations, no XCTExpectFailure).

**For each file, apply these transformations:**

1. **Imports:** Replace `import XCTest` with `import Testing`. Add `import CustomDump` if using `expectNoDifference`.
2. **Class -> Struct:** `final class FooTests: XCTestCase` -> `@Suite(.serialized) @MainActor struct FooTests`
3. **Test methods:** `func testFoo() async` -> `@Test func foo() async` (drop `test` prefix, keep lowercase start)
4. **setUp:** `override func setUp()` -> `init()` on the struct
5. **Assertions:**
   - `XCTAssertEqual(a, b)` on scalars -> `#expect(a == b)`
   - `XCTAssertEqual(a, b)` on structs/arrays -> `expectNoDifference(a, b)` (M4)
   - `XCTAssertNil(x)` -> `#expect(x == nil)`
   - `XCTAssertNotNil(x)` -> `#expect(x != nil)`
   - `XCTAssertTrue(x)` -> `#expect(x)`
   - `XCTAssertFalse(x)` -> `#expect(!x)` or `#expect(x == false)`
   - `XCTFail("msg")` -> `Issue.record("msg")` (add `return` after if needed — Issue.record does NOT stop execution)
   - `XCTAssertGreaterThan(a, b)` -> `#expect(a > b)`
6. **@MainActor:** Must be on EACH `@Suite struct` or `@Test func` that uses TestStore. Does NOT inherit through extensions.
7. **Database suites:** Add `.dependencies { try $0.bootstrapDatabase() }` trait if applicable (from Wave 3 Task 7).

**File-specific notes:**
- `SharedPersistenceTests.swift` — 15 tests, `@MainActor` needed, no expectations
- `StoreReducerTests.swift` — 11 tests, one async, CombineReducers already cleaned in Wave 2

**Build and test after EACH file migration.** Do not batch — catch errors early.
