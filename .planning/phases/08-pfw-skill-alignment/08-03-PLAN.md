  <action>
Refactor `DatabaseView` (or equivalent view in DatabaseFeature.swift) to use `@FetchAll` and `@FetchOne` property wrappers instead of the polling-via-reducer pattern, per pfw-sqlite-data canonical pattern.

**Before (polling pattern):**
```swift
// In reducer:
case .viewAppeared:
  return .run { send in
    let notes = try await database.read { ... }
    await send(.notesLoaded(notes))
  }

// In view:
.onAppear { store.send(.viewAppeared) }
```

**After (@FetchAll pattern):**
```swift
struct DatabaseView: View {
  @FetchAll(Note.order { $0.createdAt.desc() }) var notes
  @FetchOne(Note.count()) var noteCount = 0

  var body: some View {
    List(notes) { note in Text(note.title) }
  }
}
```

**Rules:**
- `@FetchAll`/`@FetchOne` are for SwiftUI views ONLY
- Remove the manual fetch Effect.run and notesLoaded action from the reducer if they become unused
- Keep any write effects (insert, delete, update) in the reducer — only reading moves to property wrappers
- For dynamic queries, initialise with `.none` and load in `.task { await $notes.load(...) }`

**Note:** If the current DatabaseFeature is heavily TCA-driven and the @FetchAll pattern would require significant architectural changes, implement it alongside the existing pattern and document what the @FetchAll version looks like. The key requirement is that the view-level observation pattern is demonstrated.

  <action>
Replace ALL `db.execute(sql: ...)` calls with the `#sql` macro per pfw-sqlite-data canonical pattern, in both production migrations and test helpers.

**Before:**
```swift
migrator.registerMigration("v1") { db in
  try db.execute(sql: """
    CREATE TABLE IF NOT EXISTS note (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      title TEXT NOT NULL DEFAULT ''
    )
    """)
}
```

**After:**
```swift
migrator.registerMigration("v1") { db in
  try #sql("""
    CREATE TABLE "note" (
      "id" INTEGER PRIMARY KEY AUTOINCREMENT NOT NULL,
      "title" TEXT NOT NULL DEFAULT ''
    ) STRICT
    """)
    .execute(db)
}
```

**DDL conventions to apply:**
- Add `STRICT` to all CREATE TABLE statements
- Quote all identifiers with double quotes
- Add `NOT NULL` on non-nullable columns (move from after type to explicit)
- Remove `IF NOT EXISTS` from migrations (migrator tracks execution)
- Keep `IF NOT EXISTS` ONLY in test helper functions that may run multiple times outside a migrator

If `#sql` genuinely cannot handle a particular DDL form (e.g., complex ALTER TABLE), document the limitation and keep raw SQL with a comment.
