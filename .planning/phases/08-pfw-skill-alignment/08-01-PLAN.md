  <name>Task 1: Replace infix operators with named query functions (H5 + M1)</name>
  <files>
    examples/fuse-library/Tests/DatabaseTests/StructuredQueriesTests.swift
    examples/fuse-library/Tests/DatabaseTests/SQLiteDataTests.swift

  <action>
**StructuredQueriesTests.swift and SQLiteDataTests.swift:**

Replace ALL infix comparison operators with named functions per pfw-structured-queries canonical pattern:
- `$0.value > 10` -> `$0.value.gt(10)`
- `$0.value == "test"` -> `$0.value.eq("test")`
- `$0.name == "nonexistent"` -> `$0.name.eq("nonexistent")`
- Any other infix `==`, `!=`, `>`, `<`, `>=`, `<=` in `.where {}` closures

Replace ALL `.asc()` calls with `order(by: \.)` key-path syntax:
- `Item.order { $0.name.asc() }` -> `Item.order(by: \.name)`
- `Item.order { $0.id.asc() }` -> `Item.order(by: \.id)`
- After a join, use closure form WITHOUT `.asc()`: `.order { items, categories in items.id }`

Keep `.desc()` calls as-is (explicit descending is correct).

Consult `/pfw-structured-queries` skill for exact patterns before applying changes. The skill docs (where.md, order-by.md) are the canonical reference.

**DO NOT change:**
- Boolean column access without operator (e.g., `$0.isActive` alone) — these are correct
- Logical operators `&&`, `||` — these are correct
- `.desc()` calls — explicit descending is fine
  </action>
  <verify>
`cd examples/fuse-library && swift test --filter StructuredQueriesTests` — all tests pass.
`cd examples/fuse-library && swift test --filter SQLiteDataTests` — all tests pass.
  </verify>
  <done>All query predicates use named functions (.eq/.gt); all redundant .asc() replaced with order(by:).</done>
</task>

  <name>Task 2: Add do/catch + reportIssue to Effect.run closures (M15)</name>
  <files>
    examples/fuse-app/Sources/FuseApp/DatabaseFeature.swift
  </files>
  <action>
Wrap ALL `Effect.run` closures in `DatabaseFeature.swift` that contain `try` with `do/catch` + `reportIssue(error)` per pfw-issue-reporting canonical pattern.

Add `import IssueReporting` if not already present.

**Before:**
```swift
return .run { send in
  let notes = try await database.read { db in
    try Note.all.order { $0.createdAt.desc() }.fetchAll(db)
  }
  await send(.notesLoaded(notes))
}
```

**After:**
```swift
return .run { send in
  do {
    let notes = try await database.read { db in
      try Note.all.order { $0.createdAt.desc() }.fetchAll(db)
    }
    await send(.notesLoaded(notes))
  } catch {
    reportIssue(error)
  }
}
```

Apply to every Effect.run closure that has a bare `try` (not already wrapped). Check all action cases in the reducer.
