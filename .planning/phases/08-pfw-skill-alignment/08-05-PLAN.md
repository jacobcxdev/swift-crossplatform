  <name>Task 3: Replace DispatchSemaphore with os_unfair_lock in bridge (LOW)</name>
  <files>
    forks/skip-android-bridge/Sources/SkipAndroidBridge/Observation.swift
  </files>
  <action>
Replace `DispatchSemaphore(value: 1)` used as a mutex in `Observation.swift` with `os_unfair_lock`, per best practice (DispatchSemaphore-as-mutex is a known anti-pattern).

**Before:**
```swift
private let semaphore = DispatchSemaphore(value: 1)
// ...
semaphore.wait()
defer { semaphore.signal() }
```

**After:**
```swift
private var lock = os_unfair_lock()
// ...
os_unfair_lock_lock(&lock)
defer { os_unfair_lock_unlock(&lock) }
```

Add `import os` if not already present (for `os_unfair_lock`).

**Android safety:** `os_unfair_lock` is available on Android via Swift Android SDK's compatibility layer. This is confirmed in the research's "Android Safety" section.

**Note:** If the lock protects JNI-facing concurrent access, ensure the replacement maintains the same mutual exclusion guarantees. `os_unfair_lock` is a strict improvement (no priority inversion, no kernel transitions for uncontended case).
  </action>
  <verify>
`cd examples/fuse-library && swift build` — compiles cleanly.
  </verify>
  <done>DispatchSemaphore replaced with os_unfair_lock for better performance and correctness.</done>
