<task type="auto">
  <name>Task 1: Fix Section 7 LocalPeerStoreItemKey SKIP INSERT pattern</name>
  <files>examples/fuse-app/Sources/FuseApp/IdentityFeature.swift</files>
  <action>
The SKIP INSERT pattern in Section 7 provides raw String literals directly to `LocalPeerStoreItemKey provides`. The CompositionLocal has type `staticCompositionLocalOf<AnyHashable?>`, and in Kotlin the `provides` infix expects `AnyHashable?`. A raw Kotlin `String` is NOT automatically `AnyHashable` — Skip's `AnyHashable` is a wrapper type. The List.swift pattern works because `itemKey` is already typed as `String` from `composeBundleNormalizedKey()` which returns String, and the SKIP INSERT assigns it to a local val that Kotlin infers as the correct type.

Fix: Wrap the String literal in the Swift variable declaration so the Skip transpiler generates the correct AnyHashable boxing. Replace the raw string SKIP INSERT with a Swift-level variable that the transpiler can type correctly.

In `IdentitySection7View.body`, replace the two SKIP INSERT blocks:

**PeerRememberTestView block — replace:**
```swift
#if SKIP
// SKIP INSERT: val providedPeerItemKey = LocalPeerStoreItemKey provides "peer-remember-test-view"
CompositionLocalProvider(providedPeerItemKey) {
    PeerRememberTestView()
}
#else
PeerRememberTestView()
#endif
```

**With:**
```swift
#if SKIP
let peerRememberItemKey: String = "peer-remember-test-view"
// SKIP INSERT: val providedPeerItemKey = LocalPeerStoreItemKey provides peerRememberItemKey
CompositionLocalProvider(providedPeerItemKey) {
    PeerRememberTestView()
}
#else
PeerRememberTestView()
#endif
```

**CounterCard block — replace:**
```swift
#if SKIP
// SKIP INSERT: val providedPeerTestCardKey = LocalPeerStoreItemKey provides "peer-test-counter-card"
CompositionLocalProvider(providedPeerTestCardKey) {
    CounterCard(title: "Peer Test Card")
}
#else
CounterCard(title: "Peer Test Card")
#endif
```

**With:**
```swift
#if SKIP
let peerTestCardItemKey: String = "peer-test-counter-card"
// SKIP INSERT: val providedPeerTestCardKey = LocalPeerStoreItemKey provides peerTestCardItemKey
CompositionLocalProvider(providedPeerTestCardKey) {
    CounterCard(title: "Peer Test Card")
}
#else
CounterCard(title: "Peer Test Card")
#endif
```

The key difference: a Swift `let` variable typed as `String` gets transpiled to a Kotlin `val` that the Skip runtime can bridge to `AnyHashable` through the standard `provides` mechanism. A raw Kotlin string literal in SKIP INSERT bypasses this bridging.
