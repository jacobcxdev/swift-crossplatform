<objective>
Fix GAP-R2-3 (tab switch peer survival not working). PeerRememberTestView in Section 7 loses identity when the user switches the app-level tab away from Identity and back.

**Root cause analysis (from RESEARCH.md):** Three hypotheses:
- **Hypothesis A (LIKELY):** The app-level TabView (Basics/Identity/etc) uses NavHost internally but does NOT have PeerStore integration. Plan 10 only added PeerStore to Section 5's mini-TabView. When the user switches app-level tabs, the entire Identity tab composition is destroyed, including all `remember`-scoped state.
- **Hypothesis B (LIKELY):** PeerRememberTestView is a standalone view not inside a ForEach. `rememberViewPeer()` requires `LocalPeerStoreItemKey` to be non-nil for the PeerStore path to activate. Without an IdentityKeyModifier (which ForEach provides), `itemKey` is nil and the fallback `remember`-based path is used, which doesn't survive composition disposal.
- **Hypothesis C:** NavHost recomposition may not preserve CompositionLocal state.

**Fix strategy:** Address both Hypotheses A and B:
1. Ensure the app-level TabView has PeerStore integration (if it uses the skip-ui TabView component, Plan 10 already added it — verify)
2. Provide a manual `LocalPeerStoreItemKey` for PeerRememberTestView in Section 7 (standalone view needs explicit item key)
3. Add diagnostic logging to `rememberViewPeer()` to confirm the PeerStore path is hit

Output: Tab switch peer survival working for Section 7, VIEWID-13 satisfied.
</objective>
