**RetainedAnimatedItems.swift — Fix Step 3 of sync() (lines 118-128):**

Replace the current Step 3 code:
```swift
// Step 3: Mark removed items for exit or remove immediately
for (key, item) in items {
    if !currentKeySet.contains(key) && item.visibility.targetState == true {
        if let animation {
            item.visibility.targetState = false
            item.animation = animation
        } else {
            items.remove(key)
        }
    }
}
```

With the collect-then-remove pattern (matching Step 4's existing approach):
```swift
// Step 3: Mark removed items for exit or collect for immediate removal
var immediateRemovalKeys = mutableListOf<Any>()
for (key, item) in items {
    if !currentKeySet.contains(key) && item.visibility.targetState == true {
        if let animation {
            item.visibility.targetState = false
            item.animation = animation
        } else {
            immediateRemovalKeys.add(key)
        }
    }
}
for key in immediateRemovalKeys {
    items.remove(key)
}
```

This is a 4-line change. The `items.remove(key)` inside the `for (key, item) in items` loop is moved to a second loop that iterates a separate list. Kotlin's LinkedHashMap iterator is no longer invalidated.
