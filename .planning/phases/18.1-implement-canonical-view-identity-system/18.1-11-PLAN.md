<objective>
Fix GAP-1 (Section 3 animation) definitively by replacing the broken `AnimatedContent` dual-path with per-item `AnimatedVisibility` via `RetainedAnimatedItems`.

**Root cause:** Container `idMap` checks `TagModifier(.id)` but ForEach wraps items with `TagModifier(.tag)`. Plan 08's idMap fallback was a partial fix — it made the AnimatedContent path reachable but `AnimatedContent` is fundamentally wrong for list-item transitions (it's designed for "swap entire content", not "animate individual items").

**Fix:** Replace the dual ANIMATED/NON-ANIMATED paths in VStack/HStack/ZStack with a single unified path using `RetainedAnimatedItems` — per-item `AnimatedVisibility` with `MutableTransitionState<Bool>`.

**Supersedes:** Plan 08's idMap fallback fix. The entire `idMap`/`ids`/`rememberedIds`/`newIds`/`rememberedNewIds`/`AnimatedContent` infrastructure is removed.

**Self-contained implementation spec:** `docs/identity-implementation-plan-animation.md` contains all current source code sections with exact line numbers and complete replacement code. A new session can implement every step from the plan alone.

Output: RetainedAnimatedItems shared helper in skip-ui, unified container rendering, visible per-item exit/enter animations with withAnimation.
</objective>
