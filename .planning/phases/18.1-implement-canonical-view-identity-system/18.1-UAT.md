## Gaps

### GAP-R2-1: ConcurrentModificationException crash in RetainedAnimatedItems.sync (CRITICAL)

**Severity:** P0 — app crashes on card deletion
**File:** `forks/skip-ui/Sources/SkipUI/SkipUI/Containers/RetainedAnimatedItems.swift`
**Root cause:** Step 3 of `sync()` iterates `items` (a `mutableMapOf<Any, RetainedAnimatedItem>()`) with `for (key, item) in items` and calls `items.remove(key)` inside the loop body. Kotlin's `LinkedHashMap` iterator throws `ConcurrentModificationException` when the map is mutated during iteration.
**Fix:** Collect keys to remove into a separate list, then remove them after the loop completes. Same collect-then-remove pattern already used in Step 4.
**Stack trace:**
```
java.util.ConcurrentModificationException
  at java.util.LinkedHashMap$LinkedHashIterator.nextNode(LinkedHashMap.java:1061)
  at skip.ui.RetainedAnimatedItemsState.sync$SkipUI_debug(RetainedAnimatedItems.kt:136)
  at skip.ui.VStack.Render(VStack.kt:69)
```

### GAP-R3-1: PeerStore lookup fails for visible items during LazyColumn deletion

**Severity:** P1 — identity lost on delete in lazy containers
**Location:** Section 6, `forks/skip-ui/Sources/SkipUI/SkipUI/Compose/PeerStore.swift`, List.swift
**Root cause (updated R4):** Two issues:
1. ~~Key type mismatch in `currentPeerStoreKeys()` — FIXED in Plan 15 (0 evicts now).~~
2. **`PeerNamespacePath` structural equality broken** — the composite namespace object changes identity between composition passes (`@20b9ece4` → `@312c7be7` → `@2f6fb2a0`). All prior PeerStore entries become orphaned (wrong namespace key), causing 100% MISS rate on revisit. Root cause: `PeerNamespacePath` is a Swift struct conforming to `Hashable`, but transpiled Kotlin class doesn't implement structural `equals()`/`hashCode()`, or `AnyHashable` wrapping loses structural equality.
**Fix approach:** Normalize namespace to String (same pattern as Plan 15 item key fix), or eliminate `PeerNamespacePath` in favour of a String-based composite key.
**Log evidence:** 25 MISS + 25 inserts per visit, 0 evicts, 295 HITs within same pass.
**Test:** R3-2, R4-1

### GAP-R3-2: PeerStore not surviving tab switch for mixed views (CounterCard)

**Severity:** P1 — mixed-view peer not remembered across tab switches
**Location:** Section 7, `forks/skip-ui/Sources/SkipUI/SkipUI/Compose/PeerStore.swift`, TabView.swift, IdentityFeature.swift
**Root cause (confirmed R4):** Two independent root causes:
1. **`LocalPeerStoreItemKey` not propagating** — `CompositionLocalProvider` with `// SKIP INSERT` val in Section 7 doesn't reach child views. Both `PeerRememberTestView` and `CounterCard` show `itemKey=null`, causing them to take the fallback `remember` path instead of PeerStore path.
2. **PeerStore destroyed on tab switch** — `releaseAll: count=24` fires when tab navigation disposes the composition. `remember { PeerStore() }` in TabView/List doesn't survive NavHost `popUpTo(inclusive=true)`. Even if itemKey worked, all cached peers are wiped. Need `rememberSaveable`-equivalent persistence for PeerStore, or scope PeerStore above NavHost.
**Log evidence:** itemKey=null for Section 7 views; releaseAll: count=24 on tab switch at 00:30:30.259.
**Test:** R3-3, R4-2
