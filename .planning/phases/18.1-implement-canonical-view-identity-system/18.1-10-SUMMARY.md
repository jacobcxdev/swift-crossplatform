## Decisions Made
- **SKIP INSERT pattern**: Swift parser validates syntax even inside `#if SKIP` blocks. Kotlin infix `provides` keyword causes parse errors. Solution: `// SKIP INSERT: val provided = Local provides value` then `CompositionLocalProvider(provided)`. Matches existing pattern in Navigation.swift.
- **Whole-value struct replacement in refreshPeer**: Struct properties are `let` constants, so per-property assignment in `Swift_refreshPeer` fails. For value types, replace the entire `SwiftValueTypeBox.value` with the fresh value. This is correct because refreshPeer only runs when inputsHash changes (constructor params differ), meaning SwiftUI would create a new struct anyway.
- **Centralised rememberViewPeer**: Transpiler no longer generates per-view `SwiftPeerHandle` class. All peer lifecycle management centralised in `PeerStore.swift` shared implementation.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Swift parser validates syntax inside #if SKIP blocks**
- **Found during:** Verification (iOS build)
- **Issue:** `CompositionLocalProvider(LocalX provides value)` uses Kotlin infix syntax. Swift parser rejects `provides` as unexpected even inside `#if SKIP` (which evaluates to false on iOS).
- **Fix:** Converted all `provides` calls to `// SKIP INSERT: val provided = Local provides value` + `CompositionLocalProvider(provided)` pattern
- **Files modified:** PeerStore.swift, Renderable.swift, LazyVStack.swift, LazyHStack.swift, LazyVGrid.swift, LazyHGrid.swift, TabView.swift
- **Verification:** iOS build succeeds
- **Committed in:** `38a6460`

**2. [Rule 1 - Bug] Swift_refreshPeer assigns to let properties on structs**
- **Found during:** Verification (Android build)
- **Issue:** Generated `cached_swift.value.title = fresh_swift.value.title` fails because `title` is a `let` constant on the struct
- **Fix:** For value types, replace entire boxed value (`cached_swift.value = fresh_swift.value`) instead of per-property assignment. Reference types keep per-property assignment (var properties).
- **Files modified:** KotlinBridgeToKotlinVisitor.swift, BridgeToKotlinTests.swift
- **Verification:** 139/139 transpiler tests pass, Android build succeeds
- **Committed in:** `ad5bb6e3`
