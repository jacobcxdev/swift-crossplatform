  - Body: a colored button that increments `tapCount`, displaying current count
- Add a label explicitly showing "Test: @State + let-with-default (no constructor params)"
- The existing `CounterCard` is retained as a SEPARATE subsection showing the "mixed view with Store" pattern
- **Manual test (Android):** Tap the `PeerRememberTestView` button 3 times. Then trigger parent recomposition (e.g., add a new card above). Verify `tapCount` is still 3 — without peer remembering, `@State` resets to 0 every recomposition on Android.
- **NOTE:** This section is verified via Android integration test ONLY — TCA TestStore cannot test transpiler/Compose peer remembering behavior. No TestStore test is created for this section.
