Close GAP-2 and GAP-3 from UAT by updating test expectations and adding VIEWID-03 through VIEWID-11 requirements to REQUIREMENTS.md.

GAP-2 (Lazy Container Identity): Compose `LazyColumn`/`LazyRow` dispose `remember{}` state when items scroll off-screen. This is expected Compose behaviour — `composeBundleNormalizedKey` provides stable diff keys but cannot persist state across disposal. The idiomatic fix is state lifting to the parent reducer (standard TCA pattern). Update UAT Test 6 expectation to reflect this platform difference.

GAP-3 (Transpiler Peer Remembering on Tab Switch): Tab switching destroys the entire tab's composition scope, causing all `remember(key)` calls to be forgotten. This is expected Compose behaviour — `remember` is scoped to the composition node. The identity system is working correctly within its scope. Update UAT Test 7 expectation to document this as a known platform difference.
