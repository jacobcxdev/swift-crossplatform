- Replaced AnimatedContent dual-path with per-item AnimatedVisibility -- the root cause was ForEach wrapping items with TagModifier(.tag) while containers checked TagModifier(.id), so the ANIMATED path was never reachable for ForEach items
- Axis-aware default transitions provide natural animations: vertical containers use fade+shrink vertically, horizontal use fade+shrink horizontally, overlay uses fade only
- Baseline-aware initial visibility (hasEstablishedBaseline) prevents the initial population of items from animating in
