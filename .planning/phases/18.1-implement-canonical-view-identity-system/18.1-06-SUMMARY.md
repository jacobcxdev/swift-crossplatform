## Decisions Made
- **explicitResetKey deferred:** The nested key-group approach works correctly for animated containers. AnimatedContent's contentKey drives animation diff (which items entered/exited), while identityKey controls positional stability within the content lambda. No need for a third field.
- **Outcome A confirmed:** key() wrapping inside AnimatedContent content lambda does not break animateEnterExit. The AnimatedVisibilityScope receiver remains available inside key() blocks, as key() only creates a movable group -- it does not alter the scope chain.
- **Spacing outside key() scope:** In animated paths, adaptive spacing is now emitted via EmitAdaptiveSpacing before the key() block, matching the non-animated loop pattern. This ensures spacing elements are not part of the keyed composition group.
