<objective>
Fix GAP-2 (lazy container disposal) and GAP-3 (tab switch composition scope destruction) — previously classified as "expected Compose platform differences" but now addressed with PeerStore: a parent-scoped peer cache that outlives individual item compositions.

**Root cause:** `remember { SwiftPeerHandle(...) }` doesn't survive Compose composition disposal. LazyColumn disposes off-screen items; TabView's NavHost with `popUpTo(saveState: true)` disposes tab compositions. `SwiftPeerHandle.onForgotten()` releases the peer, destroying let-with-default properties (Store, instanceID).

**Fix:** PeerStore (parent-scoped cache) + `rememberViewPeer()` runtime helper + transpiler mechanical replacement of `remember { SwiftPeerHandle }` with `rememberViewPeer()`.

**Self-contained implementation spec:** `docs/identity-implementation-plan-peerstore.md` contains all current source code sections with exact line numbers and insertion points. A new session can implement every step from the plan alone.

Output: PeerStore runtime infrastructure in skip-ui, transpiler changes in skipstone, peer survival across LazyColumn scroll and TabView tab switch.
</objective>
