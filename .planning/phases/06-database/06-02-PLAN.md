**Observation test pattern (from research R2 and D5):**

The `@FetchAll`/`@FetchOne`/`@Fetch` property wrappers are backed by `SharedReader` from swift-sharing. To test observation without SwiftUI views, test through `FetchKey` and GRDB's `ValueObservation` directly:

```swift
func testFetchAllObservation() async throws {
    let db = try DatabaseQueue()
    try setupSchema(db)

    // Start a ValueObservation tracking Items
    let observation = ValueObservation.tracking { db in
        try Item.fetchAll(db)
    }

    let expectation = XCTestExpectation(description: "observation triggers")
    var observedValues: [[Item]] = []

    let cancellable = observation.start(in: db, onError: { _ in }, onChange: { items in
        observedValues.append(items)
        if observedValues.count >= 2 {
            expectation.fulfill()
        }
    })

    // Initial value should be empty array
    try await Task.sleep(for: .milliseconds(100))

    // Mutate database — should trigger onChange
    try db.write { db in
        try Item.insert { $0 in
            $0.name = "alpha"
            $0.value = 42
        }.execute(db)
    }

    await fulfillment(of: [expectation], timeout: 5.0)
    cancellable.cancel()

    XCTAssertGreaterThanOrEqual(observedValues.count, 2)
    XCTAssertEqual(observedValues.first?.count, 0) // initial empty
    XCTAssertEqual(observedValues.last?.count, 1)  // after insert
}
```

For `testFetchOneObservation` — same pattern but with `ValueObservation.tracking { try Item.find(1).fetchOne($0) }`.

For `testFetchCompositeObservation` — use `ValueObservation.tracking { db in }` with multiple queries inside the tracking block (e.g., fetch count + fetch filtered items), verifying both parts update.

```swift
func testDefaultDatabaseDependency() throws {
    let testDB = try DatabaseQueue()
    try setupSchema(testDB)

    withDependencies {
        $0.defaultDatabase = testDB
    } operation: {
        @Dependency(\.defaultDatabase) var database
        let count = try database.read { db in
            try Item.fetchCount(db)
        }
        XCTAssertEqual(count, 0)
    }
}
```

**Important:** The `defaultDatabase` dependency key is defined in sqlite-data's `DefaultDatabase.swift`. Import `Dependencies` and `SQLiteData` to access it. Use `withDependencies` override pattern (established in Phase 3 DependencyTests).
